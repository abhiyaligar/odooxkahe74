# Database Models: Mini ERP

## 1. PostgreSQL (Relational Models via SQLAlchemy)

### Users
- `id`: UUID (Primary Key)
- `name`: String
- `email`: String (Unique)
- `password_hash`: String
- `role`: Enum (SuperAdmin, StoreAdmin, SalesUser, PurchaseUser, ManufacturingUser, InventoryManager, BusinessOwner)
- `is_active`: Boolean
- `created_at`: DateTime

### Products (Finished Goods & Components)
- `id`: UUID (Primary Key)
- `name`: String
- `type`: Enum (FinishedGood, Component)
- `sales_price`: Numeric
- `cost_price`: Numeric
- `on_hand_qty`: Numeric (Default 0)
- `reserved_qty`: Numeric (Default 0)
- `procurement_strategy`: Enum (MTS, MTO)
- `procure_on_demand`: Boolean
- `procurement_type`: Enum (Purchase, Manufacturing) - Nullable
- `vendor_id`: UUID (Foreign Key -> Vendors) - Nullable
- `bom_id`: UUID (Foreign Key -> BoMs) - Nullable
- `created_at`, `updated_at`: DateTime

*(Note: `free_to_use_qty` is computed on the fly as `on_hand_qty - reserved_qty` and is not stored.)*

### BoM (Bill of Materials)
- `id`: UUID (Primary Key)
- `product_id`: UUID (Foreign Key -> Products)
- `name`: String
- `version`: String
- `created_at`: DateTime

### BoMLines
- `id`: UUID (Primary Key)
- `bom_id`: UUID (Foreign Key -> BoMs)
- `component_product_id`: UUID (Foreign Key -> Products)
- `quantity_required`: Numeric

### BoMOperations
- `id`: UUID (Primary Key)
- `bom_id`: UUID (Foreign Key -> BoMs)
- `operation_name`: String
- `sequence`: Integer
- `duration_minutes`: Integer
- `work_center_id`: UUID (Foreign Key -> WorkCenters)

### WorkCenters
- `id`: UUID (Primary Key)
- `name`: String

### Vendors & Customers
- **Vendor**: `id`, `name`, `email`, `phone`, `created_at`
- **Customer**: `id`, `name`, `email`, `phone`

### Sales Orders & Lines
- **SalesOrder**: `id`, `order_number` (Unique), `customer_id` (FK), `status` (Draft, Confirmed, PartiallyDelivered, FullyDelivered, Cancelled), `created_by` (FK), `created_at`, `confirmed_at`, `delivered_at`
- **SalesOrderLine**: `id`, `sales_order_id` (FK), `product_id` (FK), `quantity_ordered`, `quantity_delivered`, `unit_price`

### Purchase Orders & Lines
- **PurchaseOrder**: `id`, `order_number` (Unique), `vendor_id` (FK), `status` (Draft, Confirmed, PartiallyReceived, FullyReceived, Cancelled), `source` (Manual, AutoGenerated), `created_by` (FK), `created_at`, `confirmed_at`, `received_at`
- **PurchaseOrderLine**: `id`, `purchase_order_id` (FK), `product_id` (FK), `quantity_ordered`, `quantity_received`, `unit_cost`

### Manufacturing Orders & Work Orders
- **ManufacturingOrder**: `id`, `order_number` (Unique), `product_id` (FK), `bom_id` (FK), `quantity_to_produce`, `status` (Draft, InProgress, Completed, Cancelled), `source` (Manual, AutoGenerated), `assignee_id` (FK), `created_at`, `started_at`, `completed_at`
- **WorkOrder**: `id`, `manufacturing_order_id` (FK), `operation_name`, `sequence`, `work_center_id` (FK), `status` (Pending, InProgress, Done), `started_at`, `completed_at`

### Stock Ledger
- `id`: UUID (Primary Key)
- `product_id`: UUID (Foreign Key -> Products)
- `change_qty`: Numeric
- `reason`: Enum (SaleDelivery, PurchaseReceipt, ManufacturingConsume, ManufacturingProduce, ManualAdjustment)
- `reference_type`: Enum (SalesOrder, PurchaseOrder, ManufacturingOrder, Manual)
- `reference_id`: UUID
- `resulting_on_hand_qty`: Numeric
- `created_at`: DateTime
- `created_by`: UUID (Foreign Key -> Users) - Nullable

## 2. MongoDB (Document Models)

### Audit Logs (Collection: `audit_logs`)
- `_id`: ObjectId
- `entity_type`: String (e.g., "SalesOrder", "Product")
- `entity_id`: UUID (References PostgreSQL ID)
- `action`: String (e.g., "StatusChanged", "PriceUpdated")
- `old_value`: Dict/JSON (Flexible schema representing old state)
- `new_value`: Dict/JSON (Flexible schema representing new state)
- `performed_by`: UUID (References PostgreSQL User ID)
- `performed_at`: ISODate Timestamp
